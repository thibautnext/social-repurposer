# Social Repurposer — SaaS MVP

Transform any article into 10+ content variants optimized for Twitter, LinkedIn, TikTok, Instagram, Facebook, and email.

## 🚀 Features

- **Multi-platform content generation**: 1 article → 50+ pieces of content
- **AI-powered optimization**: Each platform gets contextually tuned variants
- **Copy-to-clipboard ready**: One-click copy for all variants
- **URL + text input**: Paste a URL or text directly
- **JWT authentication**: Secure user accounts
- **Database ready**: PostgreSQL schema for users, articles, variants

## 📋 Tech Stack

- **Frontend**: Next.js 14 + React + Tailwind CSS
- **Backend**: Next.js API routes
- **Database**: PostgreSQL + PostgREST
- **AI**: Anthropic Claude 3.5 Sonnet
- **Auth**: JWT (bcrypt)
- **Payment**: Stripe (test mode ready)
- **Deployment**: Vercel-optimized

## ⚡ Quick Start

### 1. Clone & Install

```bash
cd /Users/thibaut/clawd/nightly-builds/social-repurposer
npm install
```

### 2. Setup Environment Variables

```bash
cp .env.example .env.local
```

Fill in the following:
- `ANTHROPIC_API_KEY`: Get from [Anthropic Dashboard](https://console.anthropic.com)
- `JWT_SECRET`: Generate with `openssl rand -base64 32`
- `POSTGREST_URL`: https://supabase.novalys.io (or your NAS PostgREST)
- `POSTGREST_ANON_KEY`: From Supabase dashboard
- `STRIPE_PUBLIC_KEY` & `STRIPE_SECRET_KEY`: From [Stripe Dashboard](https://dashboard.stripe.com)

### 3. Create Database Tables

```bash
# SSH into NAS
ssh thibaut@192.168.1.4

# Create tables
sudo docker exec nas-postgres psql -U thibaut -d main << 'EOF'
CREATE TABLE IF NOT EXISTS users (
  id SERIAL PRIMARY KEY,
  email VARCHAR(255) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE IF NOT EXISTS articles (
  id SERIAL PRIMARY KEY,
  user_id INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  url VARCHAR(2048),
  content TEXT NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE IF NOT EXISTS variants (
  id SERIAL PRIMARY KEY,
  article_id INTEGER NOT NULL REFERENCES articles(id) ON DELETE CASCADE,
  platform VARCHAR(50) NOT NULL,
  content JSONB NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE IF NOT EXISTS integrations (
  id SERIAL PRIMARY KEY,
  user_id INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  platform VARCHAR(50) NOT NULL,
  access_token VARCHAR(1000),
  refresh_token VARCHAR(1000),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
EOF

# Restart PostgREST to apply schema
sudo docker restart nas-postgrest
```

### 4. Run Development Server

```bash
npm run dev
```

Visit `http://localhost:3000`

### 5. Test Authentication

1. Sign up at `http://localhost:3000/auth/signup`
2. Use any email and password
3. You're redirected to `/dashboard`

### 6. Test Content Repurposing

1. Paste an article URL or text
2. Click "Transform Article"
3. See 10+ variants generated by Claude API
4. Copy any variant to clipboard

## 🔐 API Endpoints

### Authentication
- `POST /api/auth/signup` — Create account
- `POST /api/auth/login` — Login, get JWT token

### Content Generation
- `POST /api/repurpose` — Transform article to variants
  - Required: `Authorization: Bearer <JWT_TOKEN>`
  - Body: `{ url?: string, text?: string }`
